# Инфраструктура проекта

## Окружение

- ОС: Windows 11
- Редактор: VS Code
- Просмотр: Live Server или напрямую в браузере

## Технологии

- HTML5 / CSS3 / Vanilla JS — без фреймворков и сборщиков
- Шрифты: Google Sans (запланировано, пока Segoe UI)
- Цветовая схема: тёмно-синяя (#0f1c2e базовый фон)

## Git

- Локальный репозиторий: инициализирован
- Удалённый репозиторий: не настроен (добавить позже)
- Команда для добавления remote: `git remote add origin <url>`

## Интеграция Bitrix24

### Формы заявок
Два варианта — выбрать один:

**Вариант A — Bitrix24 Web Form (проще):**
1. В Bitrix24 → CRM → Веб-формы → создать форму
2. Скопировать embed-код и вставить в index.html
3. Лиды автоматически попадают в CRM

**Вариант B — REST webhook (гибче):**
1. В Bitrix24 → Разработчикам → Входящий вебхук
2. URL вебхука → в .env → BITRIX24_WEBHOOK_URL
3. JS делает POST fetch на этот URL при отправке формы
4. Пример запроса: `crm.lead.add` с полями имя/телефон/задача

### Чат Open Lines
1. В Bitrix24 → Контакт-центр → Открытые линии
2. Виджет → Установить на сайт → скопировать скрипт
3. Вставить перед `</body>` в index.html
4. ID виджета → в .env → BITRIX24_CHAT_WIDGET_ID

## Аналитика

### Яндекс.Метрика
1. Зарегистрировать счётчик на metrika.yandex.ru
2. Вставить код счётчика в `<head>`
3. Настроить цели:
   - Отправка формы (событие form_submit)
   - Клик по номеру телефона
   - Открытие чата
   - Скролл до секции «Услуги»

### Google Analytics 4 (опционально)
- Нужен если планируется Google Ads
- Measurement ID → в .env → GA4_MEASUREMENT_ID

## Хостинг (выбрать перед запуском)

Варианты для статического сайта:
- **Timeweb / Beget** — недорого, российский хостинг, важно для SEO
- **GitHub Pages** — бесплатно, но домен .github.io не очень для B2B
- **Собственный сервер** — если уже есть инфраструктура

## Домен и SSL

- Домен: выбрать и зарегистрировать (рекомендую .ru для SEO)
- SSL: обязательно (Let's Encrypt бесплатно, или от хостинга)
- Редирект www → без www (или наоборот) — настроить однократно

## Переменные окружения

Все ключи хранятся в `.env` (не в Git). Шаблон — `.env.example`.
Поскольку сайт статический, ключи используются только при разработке
или вшиваются в HTML вручную перед деплоем (Bitrix24 webhook URL — не секрет
в полном смысле, его можно вставить напрямую в JS).
